**文档版本**：v1.0  
**更新日期**：2025-03-13  
**状态**：初稿

---

## 一、产品概述

### 1.1 产品名称

技能五子棋（网页版）

### 1.2 产品定位

一款基于经典五子棋规则、加入「技能卡」机制的网页对战游戏。支持双人单机与单人 vs 本地 AI 两种模式，后续可扩展联机对战与小程序端。

### 1.3 目标用户

- **核心用户**：喜欢策略与随机性结合的棋类玩家、轻度休闲玩家。
- **使用场景**：浏览器内即开即玩，无需安装；单机可双人同屏或单人挑战 AI。

### 1.4 核心价值

- **规则易懂**：五子连珠 + 技能卡，上手快。
- **策略与运气并存**：落子积累技能点，抽卡带来变数，增加重玩价值。
- **多端预留**：首版为网页，架构预留联机与小程序扩展。

## 二、游戏规则（定稿）

以下规则在 PRD 中写死，实现时以本文档为准，避免歧义。

### 2.1 基础规则

| 项目 | 说明 |
|------|------|
| 棋盘 | 标准 15×15（或 19×19，首版建议 15×15） |
| 胜利条件 | 横、竖、斜任意方向先形成五子连珠者胜 |
| 大对局 | 三局两胜制；先赢得 2 小局的一方获胜 |

### 2.2 先手规则

- **第 1 小局**：黑方先手。
- **后续小局**：采用「上一局败者先手」或「固定黑先」二选一（产品确定后写入规则说明）。

### 2.3 和棋

- 当某小局棋盘下满且无人五连时，判该小局**和棋**。
- 和棋不记分，本局重开或双方 0:0 进入下一小局（二选一统一）。

### 2.4 回合与技能点

- 每回合二选一：**落子** 或 **抽技能卡**。
- **落子**：在空交叉点下一子，且本回合获得 **1 技能点**。
- **抽技能卡**：消耗 **3 技能点**，从牌堆抽一张卡并**必须当场使用**，不可保留。
- 技能点按玩家分别累计，不跨小局继承（新小局重新从 0 开始，除非特别说明）。

### 2.5 技能卡与效果（9 张）

| 技能名 | 效果摘要 | 规则补充 |
|--------|----------|----------|
| 飞沙走石 | 移除对方一子 | 需先选对方一子，确认后从棋盘移除 |
| 静如止水 | 对方跳过下一轮 | 下一回合对方自动跳过 |
| 力拔山兮 | 清空棋盘，技能点保留 | 清盘后**对方**下一手先下，避免连续清盘拖局 |
| 调你离山 | 移动对方一子到空位 | 先选对方一子，再选空位，确认后移动 |
| 两极反转 | 双方颜色互换 | 只改「谁执黑/执白」映射，棋子不动；**发动技能的玩家**继续行动（用对方颜色下一子） |
| See You Again | 移除对方全部子 | 遍历棋盘，删除所有对方颜色的子 |
| 大获全胜 | 本局直接胜利 | 当前玩家大比分 +1，本局立即结束；若已 2 胜则进入结算 |
| 滴水穿石 | 本回合可再下两子 | 本回合状态改为「额外下 2 子」，不消耗额外技能点 |
| 满盘皆输 | 本局直接失败 | 对方大比分 +1，本局立即结束 |

**牌堆**：9 张技能卡等概率随机抽取；首版可等概率，后续可迭代为概率加权或每局每种限抽一次。

**大获全胜 / 满盘皆输**：抽到后当前小局立即结束，大比分相应 +1；若已 2 胜则直接进入结算。结算前建议有简短提示（如「大获全胜！」）。

## 三、功能需求

### 3.1 功能模块总览

```
┌─────────────────────────────────────────────────────────┐
│  标题页（模式选择）                                        │
├─────────────────────────────────────────────────────────┤
│  游戏页（对局、棋盘、技能卡、玩家信息、回合）                  │
├─────────────────────────────────────────────────────────┤
│  结算页（大比分、胜负展示、再来一局/退出游戏）                 │
└─────────────────────────────────────────────────────────┘
```

### 3.2 标题页

| 需求 ID | 需求描述 | 优先级 |
|---------|----------|--------|
| F-Title-01 | 展示游戏名称「技能五子棋」 | P0 |
| F-Title-02 | 提供「双人单机」入口：点击后进入游戏页，双人轮流对战 | P0 |
| F-Title-03 | 提供「单人 vs AI」入口：点击后进入游戏页，玩家与本地 AI 对战，AI 使用与玩家相同的技能规则与技能卡 | P0 |
| F-Title-04 | 提供「双人在线联机」入口：灰显或占位，文案「敬请期待」，点击可 toast 提示 | P1 |
| F-Title-05 | 可选：规则说明 / 技能卡列表入口（弹窗或单独说明页） | P2 |

### 3.3 游戏页

| 需求 ID | 需求描述 | 优先级 |
|---------|----------|--------|
| F-Game-01 | **顶部**：展示 Player 1 / Player 2 名称、代表色（黑/白）、当前技能点数 | P0 |
| F-Game-02 | **中部**：展示大比分（如 1:0）；下方为标准 15×15 棋盘 | P0 |
| F-Game-03 | 点击空交叉点可落子；落子后获得 1 技能点，回合切换至对方 | P0 |
| F-Game-04 | **底部**：技能卡牌堆（牌背「技能卡」），点击 = 消耗 3 技能点并抽一张卡，立即进入该技能流程 | P0 |
| F-Game-05 | 每回合二选一：落子 **或** 抽技能卡；抽卡必须当场使用，不能保留 | P0 |
| F-Game-06 | 需选子/选点的技能（如飞沙走石、调你离山）：先选棋子/再选格，选中后轮廓高亮；底部提供「确认」「取消」；确认执行，取消可重选 | P0 |
| F-Game-07 | 判赢：每次落子或改变棋盘状态的技能后，检测横竖斜五子连珠；满足则本小局结束，更新大比分 | P0 |
| F-Game-08 | 大比分先达 2 胜时，进入结算页 | P0 |
| F-Game-09 | 大获全胜/满盘皆输触发时，结算前有简短提示（如「大获全胜！」） | P1 |

### 3.4 结算页

| 需求 ID | 需求描述 | 优先级 |
|---------|----------|--------|
| F-Settle-01 | 上方展示最终大比分 | P0 |
| F-Settle-02 | 中间：获胜方名称放大展示；败方名称小图、灰色，置于胜方右下 | P0 |
| F-Settle-03 | 底部「退出游戏」：返回标题页 | P0 |
| F-Settle-04 | 底部「再来一局」：同模式重新开局，大比分清零，三局两胜重新开始 | P0 |

## 四、流程与状态

### 4.1 页面流程

```
标题页 ──选择模式──> 游戏页 ──大比分 2 胜──> 结算页
                         ^                        │
                         │    一局结束继续         │
                         └────────────────────────┘
                                              再来一局
                         ┌────────────────────────┐
                         │ 退出游戏 ──────────────> 标题页
                         └────────────────────────┘
```

### 4.2 游戏核心状态抽象

| 概念 | 说明 |
|------|------|
| **Match** | 大对局，三局两胜；包含多小局、当前小局索引、双方大比分 |
| **Game** | 单小局；包含棋盘、当前行棋方、双方技能点、技能牌堆、本局结果（未结束/黑胜/白胜/和棋） |
| **Turn** | 当前小局内回合；状态：可落子或抽卡 / 正在执行技能（选子、选格、确认等） |

技能卡执行会改变棋盘或触发「跳过回合」「本局直接胜/负」等，需在游戏逻辑层统一封装（如 `applySkill(skillId, params)`），以便 AI 与联机校验复用同一套规则。

## 五、非功能需求

### 5.1 技术约束

| 项目 | 要求 |
|------|------|
| 平台 | 现代浏览器（Chrome / Safari / Firefox 等），支持 ES 模块 |
| 技术栈 | Vue 3 + TypeScript + Vite；样式可用 CSS Modules 或 UnoCSS/Tailwind |
| 状态管理 | 单机用 Pinia 或 组合式 + provide/inject；联机阶段再考虑服务端同步 |
| 核心逻辑 | 棋盘、判赢、技能效果等与框架解耦（纯 TS 模块），便于后续迁小程序 |

### 5.2 体验要求

| 项目 | 要求 |
|------|------|
| 响应 | 落子、抽卡、确认等操作有即时反馈 |
| 可读性 | 当前回合、当前玩家、技能点、大比分清晰可见 |
| 可选 | 音效、简单动画可在后续迭代加入 |

### 5.3 扩展预留

- **联机**：对局状态可序列化（如 MatchState + GameState 的 JSON），便于后续 WebSocket 房间同步与校验。
- **小程序**：核心逻辑无 DOM 依赖，便于用 Taro/uni-app 等迁移同一套 game/ 与 ai/ 逻辑。

## 六、单人 vs AI 需求

| 需求 ID | 需求描述 | 优先级 |
|---------|----------|--------|
| F-AI-01 | AI 与玩家使用相同规则：判赢、技能卡效果均走同一套逻辑 | P0 |
| F-AI-02 | AI 会落子：基于启发式评估（己方五/四/三连、堵对方等），可选极大极小加深 | P0 |
| F-AI-03 | AI 会使用技能：若本回合必须执行技能则按规则做合法操作；若可选落子或抽卡，可按策略与简单权重决策 | P0 |
| F-AI-04 | 首版支持一个难度；后续可扩展简单/困难（少算几层/随机多 vs 搜索加深） | P1 |

## 七、实现阶段与范围

| 阶段 | 范围 | 交付物 |
|------|------|--------|
| **阶段 1** | 基础框架 | Vue 3 + Vite + TS 项目；路由：标题页 / 游戏页 / 结算页；标题页仅「双人单机」可点 |
| **阶段 2** | 棋盘与规则 | 棋盘数据结构，落子、判五子连珠，三局两胜与一局结束、大比分、切换小局 |
| **阶段 3** | 游戏页 UI | 双人信息栏、比分、技能点、棋盘点击落子、回合切换；无技能卡，先跑通双人单机 |
| **阶段 4** | 技能系统 | 牌堆、抽卡消耗 3 点、9 种技能效果；需确认的技能用选中 + 确认/取消 |
| **阶段 5** | 结算页 | 按大比分展示胜负、再来一局/退出游戏 |
| **阶段 6** | 单人 vs AI | 本地 AI 模块（启发式 + 可选极大极小），标题页开放「单人 vs AI」 |
| **阶段 7** | 联机占位 | 标题页「双人在线联机」灰显或「敬请期待」 |
| **阶段 8** | 体验与平衡 | 可选音效/动画；规则文档定稿；技能卡概率或限制的平衡性迭代 |

## 八、验收标准（摘要）

- 双人单机可完整完成三局两胜对局，落子、技能点、抽卡、9 种技能效果符合上述规则。
- 单人 vs AI 可完整对局，AI 会合理落子并使用技能。
- 结算页正确展示胜负与再来一局/退出游戏。
- 先手、和棋、力拔山兮/两极反转等细节与本文档 2.2、2.3、2.5 一致。
